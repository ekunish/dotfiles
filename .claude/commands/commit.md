I'll help you create a git commit following our emoji-enhanced conventional commit format.

Let me analyze the current changes and create an appropriate commit following the guidelines in @claude/commit-convention.md:

1. Check git status and diff to understand what changed
2. Create a commit message with appropriate emoji and type
3. Perform the git commit

**Format**: `<emoji> <type>(<scope>): <subject>`

**Common types with emojis**:
- ✨ feat: New feature
- 🐛 fix: Bug fix
- 📝 docs: Documentation changes
- ♻️ refactor: Code refactoring
- 🎨 style: Code formatting
- ⚡ perf: Performance improvements
- ✅ test: Test updates
- 📦 build: Build/dependency changes
- 🔧 config: Configuration changes

I'll follow the full convention guide at @claude/commit-convention.md for the complete emoji list and rules.
